# 文本显示溢出问题报告

## 一、根因分析

`renderer.js:73-81` 的 `drawText()` 只调用了 Canvas 的 `ctx.fillText(text, x, y)`，该 API 绘制**单行文本**，超出容器宽度的文字直接溢出到弹窗外，无自动换行、无截断。

`uiManager.js:218-227` 渲染内容时，仅按 `\n` 拆分成行，每行直接调用 `drawText()`，从未使用 `measureText()` 检测文本宽度是否超过可用区域。

| 文件 | 行 | 问题 | 风险等级 |
|------|------|------|----------|
| `renderer.js` | 73-81 | `drawText()` 无换行/截断逻辑 | **根因** |
| `renderer.js` | 104-128 | `drawGhibliButton()` 按钮文本无截断 | 中 |
| `uiManager.js` | 218-227 | `renderModal()` 内容行无宽度检测 | 高 |
| `uiManager.js` | 347-363 | `drawGhibliButton()` 同 renderer | 中 |

---

## 二、具体溢出点

### P1 — 银行弹窗（实际溢出）

**文件**：`js\managers\backgamemanger.js:296`

```js
content: `存款利率: 每${GAME_CONFIG.bank.depositInterestDays}天${...}%  |  贷款利率: 每${GAME_CONFIG.bank.loanInterestDays}天${...}%\n当前欠款: ${state.bankLoan.toLocaleString()} 金币\n\n${jobContent}`,
```

**问题**：第一行拼接 `存款利率: 每6天2%  |  贷款利率: 每6天6%`，中文约 24 字符 ≈ **322px 宽**。弹窗可用文字宽度约 `375×0.85 - 40 ≈ 279px`，**溢出约 43px**。

**相同模式**：`js\systems\bankruptcySystem.js:36`（银行还款弹窗）。

---

### P2 — 工作弹窗（组装在同一行）

**文件**：`js\managers\backgamemanger.js:177`

```js
content: `【当前岗位】\nLv.${job.level} ${job.title}\n\n上班: ${baseSalary}金币 (消耗1精力)\n加班: ${overtimeSalary}金币 (消耗1精力,健康-5)${nextLevelInfo}`,
```

**问题**：`nextLevelInfo` 展开为 `\n\n【下一级】\nLv.x xxx\n日薪: xxx金币`，按 `\n` 拆分后各行长度约 200-230px（13px 字体），接近可用宽度 279px 但暂未溢出。若职位名较长则可能溢出。

---

### P2 — 报纸弹窗（动态拼接）

**文件**：`js\scenes\marketScene.js:111`

```js
const newspaperContent = newspaperData.map(news => `【${news.title}】\n${news.content}`).join('\n\n')
```

**问题**：报纸内容来自 `newspaperData`（动态生成），如果某条新闻的 `content` 是长中文句，报纸模式下用 14px 居中绘制无换行，超过 279px 即溢出。当前预设新闻长度可控，但若接入外部数据则风险较高。

---

### P3 — 按钮文本

**文件**：`uiManager.js:347-363`、`renderer.js:104-128`

```js
ctx.fillText(text, x + w / 2, y + h / 2)
```

**问题**：按钮文本同样无宽度检测。目前按钮宽度在弹窗中最大约 `(modalW - 40) / 2 ≈ 140px`，文本超过该宽度则溢出按钮边框。

---

## 三、解决方案

### 方案：自动换行 + 弹窗高度自适应

改动集中在 **`renderer.js`** 和 **`uiManager.js`**，不波及 Scene 层。

#### 3.1 新增 `wrapText()` 方法（renderer.js）

```js
/**
 * 将文本按最大宽度自动换行
 * @param {string} text - 原始文本（支持 \n 手动换行）
 * @param {number} maxWidth - 最大像素宽度
 * @param {number} fontSize - 字体大小
 * @returns {string[]} 换行后的行数组
 */
wrapText(text, maxWidth, fontSize = 13) {
    const ctx = this.ctx
    ctx.font = `${fontSize}px sans-serif`
    const lines = []
    const paragraphs = text.split('\n')
    for (const paragraph of paragraphs) {
        if (ctx.measureText(paragraph).width <= maxWidth) {
            lines.push(paragraph)
            continue
        }
        // 逐字符拆分
        let line = ''
        for (const char of paragraph) {
            const testLine = line + char
            if (ctx.measureText(testLine).width > maxWidth) {
                lines.push(line)
                line = char
            } else {
                line = testLine
            }
        }
        if (line) lines.push(line)
    }
    return lines
}
```

#### 3.2 修改 `renderModal()` 高度逻辑（uiManager.js）

```diff
- const modalH = modal.height || 180
+ // 自动计算弹窗高度
+ const hasContent = modal.content && modal.content.length > 0
+ let contentLines = []
+ if (hasContent) {
+     const textMaxWidth = modalW - 40  // 左右各 20px 边距
+     contentLines = renderer.wrapText(modal.content, textMaxWidth, modal.isNewspaper ? 14 : 13)
+ }
+ const contentLineCount = contentLines.length
+ const titleArea = modal.isNewspaper ? 70 : 65
+ const btnArea = 60
+ const lineHeight = 22
+ const contentArea = hasContent ? titleArea + contentLineCount * lineHeight + 15 : titleArea
+ const modalH = Math.max(180, contentArea + btnArea)
```

#### 3.3 修改内容渲染部分（uiManager.js）

```diff
- const lines = modal.content.split('\n')
+ const lines = contentLines  // 使用 wrapText 处理后的行
  let lineY = contentStartY
  for (const line of lines) {
      // ... 绘制逻辑不变 ...
      lineY += 22
  }
```

#### 3.4 修改按钮位置（uiManager.js）

按钮的 Y 坐标需要从 `modalY + modalH - 55` 改为基于计算后的 `modalH`（已通过变量引用，无需额外改动）。

## 四、总结

| 优先级 | 问题 | 状态 | 修复方式 |
|--------|------|------|----------|
| P0 | 所有文本无法自动换行 | 待修复 | 新增 `wrapText()` + 集成到 `renderModal()` |
| P1 | 银行弹窗文本溢出约 43px | 自动修复 | 替换 `\n` 拆分为 `wrapText()` 后自动断行 |
| P2 | 工作/报纸/事件弹窗潜在溢出 | 自动修复 | 同上 |
| P3 | 按钮文本无截断 | 待评估 | 可新增 `truncateText()` 或限制按钮文本长度 |

**结论**：纯新增逻辑，不修改现有绘制流程，改动风险低，兼容所有现有弹窗调用。
