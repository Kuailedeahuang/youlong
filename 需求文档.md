## 一、项目概述

### 1.1 项目背景

本游戏是一款以"大城市打拼买房娶妻"为主题的文字模拟经营类微信小程序游戏。玩家扮演一个怀揣梦想来到大城市打拼的年轻人，需要在180天内通过市场交易、工作、投资等方式积累财富，最终购买房产实现人生目标。

### 1.2 核心设计理念

**市场交易为核心赚钱途径，工作为辅助保底手段。**

- 市场操作是积累财富的主要方式
- 工作收入仅能维持基本生活，无法靠工作买房
- 高风险高回报的市场机制
- 完善的负债与破产系统增加挑战性

### 1.3 目标用户

| 用户特征 | 描述 |
|----------|------|
| 年龄层 | 18-35岁 |
| 游戏偏好 | 喜欢模拟经营、策略类游戏 |
| 游戏时间 | 碎片化时间，单局15-30分钟 |
| 平台习惯 | 微信小程序用户 |

### 1.4 游戏核心参数

| 参数 | 数值 | 说明 |
|------|------|------|
| 游戏周期 | 180天 | 玩家需要在180天内完成目标 |
| 初始金币 | 5000 | 游戏起始资金 |
| 目标房价 | 220000 | 标准一居室价格 |
| 每日精力 | 5点 | 初始精力上限，可提升 |
| 商品种类 | 38种 | 7大类商品可供交易 |

## 二、功能需求

### 2.1 核心游戏机制

#### 2.1.1 每日循环系统

**需求描述：** 游戏以"天"为单位推进，每天玩家可执行有限次数的行动。

**功能要点：**
- 每天开始自动扣除固定生活消费
- 刷新当日报纸（预测明日行情）
- 触发随机事件
- 玩家消耗精力执行行动
- 结束今日后进入下一天

**每日消费规则：**
| 消费项目 | 金额 | 说明 |
|----------|------|------|
| 吃饭+交通 | 30金币 | 必须消费 |
| 郊区住房 | 50金币 | 默认住房 |
| 市区住房 | 100金币 | 更高消费 |
| 已买房 | 20金币 | 物业费 |

**金币不足处理：**
- 自动借入私人借贷补足差额
- 借款金额额外收取5%罚息

#### 2.1.2 精力系统

**需求描述：** 精力是玩家执行行动的核心资源。

**功能要点：**
- 初始精力上限：5点
- 每日恢复至上限
- 不同行动消耗不同精力
- 精力上限可通过健身提升

**精力消耗表：**
| 行动 | 精力消耗 |
|------|----------|
| 市场交易 | 2点 |
| 工作/加班 | 1点 |
| 健身 | 1点 |
| 医院 | 1点 |
| 娱乐 | 1点 |
| 银行操作 | 0点 |
| 借贷操作 | 0点 |
| 购房 | 1点 |

**精力上限影响因素：**
- 健康值 < 50：精力上限 -2
- 健康值 < 30：精力上限再 -2
- 连续健身 ≥ 3天：精力上限 +1
- 连续健身 ≥ 7天：精力上限再 +1

### 2.2 市场交易系统（核心）

#### 2.2.1 商品系统

**需求描述：** 市场提供38种商品供玩家交易，分为7大类。

**商品分类及波动特性：**

| 分类 | 商品数量 | 自然波动范围 | 示例商品 |
|------|----------|--------------|----------|
| 农副产品 | 10种 | ±5%~±8% | 大米、猪肉、牛肉、鸡蛋 |
| 工业建材 | 5种 | ±10%~±15% | 钢材、水泥、玻璃 |
| 贵金属奢侈品 | 4种 | ±12%~±18% | 黄金、白银、珠宝原石 |
| 能源化工 | 4种 | ±10%~±15% | 煤炭、汽油、天然气 |
| 日用百货 | 5种 | ±5%~±7% | 布料、日化用品 |
| 数码产品 | 5种 | ±15%~±20% | 酷噶手机、鸭梨平板 |
| 出行工具 | 5种 | ±12%~±18% | 电动自行车、二手汽车 |

**价格限制规则：**
- 最低价格：基础价格 × 10%
- 最高价格：基础价格 × 300%

#### 2.2.2 价格波动机制

**需求描述：** 商品价格每日波动，受自然波动和新闻事件影响。

**自然波动规则：**
- 每种商品有固定的自然波动范围
- 每日随机在范围内波动
- 波动独立于其他商品

**新闻事件影响：**
- 报纸预测明日特定商品的价格变化
- 新闻影响幅度：±20%~±50%
- 新闻影响叠加自然波动

**价格波动示例：**
```
黄金基础价格：200
自然波动范围：±18%
今日新闻：黄金需求大涨 +30%

计算：
自然波动随机值：+10%
新闻影响：+30%
最终价格 = 200 × (1 + 0.10) × (1 + 0.30) = 286
```

#### 2.2.3 交易操作

**需求描述：** 玩家可在市场买入或卖出商品。

**买入规则：**
- 消耗2精力
- 金币足够支付总价
- 仓库有足够容量
- 单次可购买多个同类商品

**卖出规则：**
- 消耗2精力
- 仓库中有该商品
- 按当前市场价出售
- 单次可出售多个同类商品

**仓库系统：**
- 初始容量：20格
- 每种商品占用1格（不限数量）
- 可通过升级扩大容量

### 2.3 报纸系统

#### 2.3.1 报纸功能

**需求描述：** 报纸是玩家预测市场行情的核心工具。

**功能要点：**
- 每日自动刷新
- 显示3-5条新闻
- 每条新闻关联特定商品
- 预测明日价格变化方向和幅度

**新闻示例：**
```
【农业快讯】南方洪灾影响农作物收成，预计大米、蔬菜价格将上涨25%-40%
【科技动态】新款手机发布，酷噶手机、鸭梨平板可能降价15%-25%
【能源报告】国际油价波动，汽油价格预计上涨30%-50%
```

### 2.4 工作系统

#### 2.4.1 职位等级

**需求描述：** 玩家可通过工作获得稳定收入，但收入有限。

**职位等级表：**

| 等级 | 职位名称 | 上班收入 | 加班收入 | 每日满勤 | 银行贷款额度 |
|------|----------|----------|----------|----------|--------------|
| 1 | 外卖/快递员 | 120 | 180 | 420 | 50,000 |
| 2 | 文员 | 240 | 360 | 840 | 50,000 |
| 3 | 主管 | 480 | 720 | 1,680 | 75,000 |
| 4 | 经理 | 960 | 1,440 | 3,360 | 75,000 |
| 5 | 总监 | 1,920 | 2,880 | 6,720 | 75,000 |

**设计说明：** 即使升到最高职位，每日满勤收入仅6720金币，180天总收入约120万，扣除生活消费后难以买房，必须依靠市场交易。

#### 2.4.2 工作操作

**上班：**
- 消耗1精力
- 获得基础薪资
- 每天最多上班3次

**加班：**
- 消耗1精力
- 健康值 ≥ 30 才可加班
- 获得1.5倍薪资
- 健康 -5
- 每天最多加班2次

**降薪状态：**
- 触发降薪事件后，收入 × 0.8
- 持续7天
- 期间可再次触发降薪

**失业状态：**
- 触发失业事件后，无法工作
- 持续3天
- 失业期间无法获得任何工作收入

#### 2.4.3 职场变动

**升职：**
- 基础概率：5%
- 健康值 ≥ 80 且满勤：+15%
- 名誉值 ≥ 70：+5%
- 最高概率：25%

**降薪：**
- 概率：4%
- 触发后收入打8折
- 持续7天

**失业：**
- 概率：3%
- 触发后3天无法工作

### 2.5 银行系统

#### 2.5.1 存款功能

**需求描述：** 玩家可将金币存入银行获取利息。

**功能要点：**
- 存款利率：每6天2%
- 随时存取
- 无手续费

#### 2.5.2 贷款功能

**需求描述：** 玩家可向银行申请贷款。

**功能要点：**
- 贷款额度：根据职位等级决定
- 贷款利率：每6天6%
- 随时可还款
- 贷款未还清无法购房

**贷款额度：**
| 职位等级 | 贷款额度 |
|----------|----------|
| 1-2级 | 50,000 |
| 3-5级 | 75,000 |

### 2.6 借贷系统（高利贷）

#### 2.6.1 私人借贷

**需求描述：** 玩家可向私人借贷获取资金，但风险极高。

**功能要点：**
- 无额度限制
- 利率：每日2.5%
- 逾期罚息：每日5%
- 逾期7天触发破产

**风险机制：**
- 金币不足支付消费时自动借入
- 自动借入部分额外收取5%罚息
- 高利贷未还清无法购房

### 2.7 破产系统

#### 2.7.1 破产判定

**需求描述：** 当玩家财务状况恶化到一定程度时触发破产。

**破产触发条件（满足任一）：**
1. 总负债 > 总资产 × 1.5 且逾期天数 ≥ 2天
2. 健康值 ≤ 0 且金币 < 50
3. 私人借贷逾期 ≥ 7天

**总资产计算：**
```
总资产 = 金币 + 银行存款 + 仓库商品估值
```

#### 2.7.2 破产清算

**破产后果：**
- 仓库清空
- 银行存款清零
- 金币重置为1000
- 职位降为1级
- 所有贷款清零
- 破产次数 +1

### 2.8 健康系统

#### 2.8.1 健康值管理

**需求描述：** 健康值影响精力上限和生病概率。

**健康值范围：** 0-100

**健康值影响因素：**
| 行动/事件 | 健康变化 |
|-----------|----------|
| 加班 | -5 |
| 健身 | +5~+10 |
| 生病（轻症） | -15~-25 |
| 生病（重症） | -35~-50 |
| 医院（轻症） | 治愈 |
| 医院（重症） | 治愈 |

#### 2.8.2 生病机制

**生病概率：**
| 健康值 | 生病概率 | 病情类型 |
|--------|----------|----------|
| ≥ 90 | 0% | 无 |
| 70-89 | 5% | 轻症 |
| 40-69 | 20% | 70%轻症/30%重症 |
| < 40 | 50% | 20%轻症/80%重症 |

**生病后果：**
- 轻症：健康 -15~-25，可自愈或去医院
- 重症：健康 -35~-50，必须去医院

#### 2.8.3 健身系统

**需求描述：** 健身可提升健康值和精力上限。

**健身效果：**
- 消耗1精力
- 健康值 +5~+10
- 连续健身天数 +1

**连续健身奖励：**
| 连续天数 | 精力上限加成 |
|----------|--------------|
| ≥ 3天 | +1 |
| ≥ 7天 | +2 |

**中断规则：** 连续健身天数归零，重新计算

### 2.9 心情与名誉系统

#### 2.9.1 心情系统

**需求描述：** 心情值影响健康值的自然衰减。

**心情值范围：** 0-100

**心情影响因素：**
| 行动/事件 | 心情变化 |
|-----------|----------|
| 娱乐 | +15~+25 |
| 公益 | +5~+10 |
| 负面事件 | -10~-20 |
| 破产 | -30 |

**心情对健康的影响：**
- 心情 < 30：每日健康 -3
- 心情 < 50：每日健康 -1

#### 2.9.2 名誉系统

**需求描述：** 名誉值影响结局判定和职场发展。

**名誉值范围：** 0-100（可为负）

**名誉影响因素：**
| 行动/事件 | 名誉变化 |
|-----------|----------|
| 公益 | +5~+10 |
| 破产 | -20 |
| 逾期还款 | -5 |
| 负面事件 | -10~-15 |

**名誉影响：**
- 名誉 ≥ 70：升职概率 +5%
- 名誉 < 0：触发"名誉扫地"结局
- 名誉 < 20 且健康 < 40：触发"怨偶"结局

### 2.10 广告系统

#### 2.10.1 激励视频广告

**需求描述：** 玩家可通过观看广告获取金币。

**功能要点：**
- 每次观看获得5000金币
- 最多观看3次
- 全局限制，非每日限制

**设计目的：** 为陷入困境的玩家提供一次性的资金援助，同时增加游戏变现渠道。

### 2.11 购房系统

#### 2.11.1 购房条件

**需求描述：** 购房是游戏的最终目标。

**购房条件：**
1. 银行贷款 = 0
2. 私人借贷 = 0
3. 现金 ≥ 房价（220,000）
4. 天数 ≤ 180

**房产信息：**
| 房产名称 | 价格 | 父母态度 |
|----------|------|----------|
| 标准一居室 | 220,000 | 勉强认可 |

#### 2.11.2 购房后效果

- 住房消费降为20金币/天
- 触发结局判定

### 2.12 存档系统

#### 2.12.1 存档功能

**需求描述：** 支持游戏进度保存和读取。

**功能要点：**
- 自动存档：每日结束时
- 手动存档：存档页面操作
- 多存档位：支持3个存档位
- 云存档：可选的云同步功能

**存档内容：**
- 玩家状态（金币、属性、天数等）
- 仓库物品
- 市场状态
- 新闻历史

## 三、结局系统

### 3.1 结局类型

| 结局名称 | 触发条件 | 描述 |
|----------|----------|------|
| 圆满成功 | 购房且无贷款且名誉≥20且健康≥40 | 成功买房娶妻，人生赢家 |
| 怨偶 | 购房且(名誉<20或健康<40) | 买房了但身心俱疲，婚姻不幸福 |
| 遗憾分手 | 购房但有贷款 | 买房了但背负债务，对象离开 |
| 时间耗尽 | 180天未购房 | 时间到了，没能买房 |
| 健康崩溃 | 健康值 ≤ 0 | 过度劳累，身体垮了 |
| 名誉扫地 | 名誉值 < 0 | 名声太差，无法立足 |
| 破产清算 | 触发破产 | 负债累累，一无所有 |

### 3.2 结局判定优先级

1. 健康崩溃
2. 名誉扫地
3. 破产清算
4. 时间耗尽
5. 购房相关结局（圆满成功/怨偶/遗憾分手）

## 四、随机事件系统

### 4.1 事件分类

| 类别 | 触发概率 | 示例 |
|------|----------|------|
| 市场事件 | 30% | 某商品价格异常波动 |
| 职场事件 | 25% | 升职/降薪/失业 |
| 生活事件 | 25% | 意外支出/额外收入 |
| 健康事件 | 20% | 生病/意外受伤 |

### 4.2 事件示例

**正面事件：**
- 中奖：获得500-2000金币
- 朋友借款归还：获得1000-3000金币
- 健身效果翻倍：今日健身效果×2

**负面事件：**
- 手机损坏：损失300-800金币
- 生病：健康值下降
- 交通事故：损失500-1500金币

**中性事件：**
- 市场传言：某商品价格将波动
- 政策变化：影响特定商品

### 5.3 交互设计

**按钮反馈：**
- 点击时震动反馈
- 按钮按下变色
