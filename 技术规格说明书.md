## 一、系统架构概述

### 1.1 技术栈选型

| 层级 | 技术方案 | 说明 |
|------|----------|------|
| 游戏引擎 | 原生 Canvas 2D | 自研渲染引擎，无第三方框架依赖 |
| 渲染方式 | CanvasRenderingContext2D | 双缓冲渲染，支持高清分辨率 |
| 状态管理 | GameState 单例模式 | 集中管理游戏状态，支持云端+本地双存档 |
| 数据存储 | 云数据库 + localStorage | 云端同步存档，用户解锁房屋永久保留 |
| 场景管理 | SceneManager 场景管理器 | 基于场景的模块化架构 |
| UI 系统 | UIManager 组件化UI | 弹窗、按钮、文本等UI组件统一管理 |
| 动画系统 | AnimationManager 延迟动画队列 | 属性变化动画延迟播放，避免卡顿 |

## 三、核心算法规格

### 3.1 每日自然价格波动算法

```javascript
function initPrices() {
    ItemData.forEach(item => {
        const fluctuation = item.naturalFluctuation.min +
            Math.random() * (item.naturalFluctuation.max - item.naturalFluctuation.min)
        this.itemPrices[item.id] = Math.round(item.basePrice * (1 + fluctuation))
    })
}
```

### 3.2 精力上限计算

```javascript
function calculateMaxEnergy(baseMax, health, consecutiveGymDays) {
    let maxEnergy = baseMax

    if (consecutiveGymDays >= 3) {
        maxEnergy = Math.min(15, maxEnergy + 1)
    }

    return Math.max(1, Math.min(15, maxEnergy))
}
```

### 3.3 利息结算算法

```javascript
function settleInterest(playerState) {
    if (playerState.day % 6 === 0) {
        playerState.bankDeposit = Math.floor(playerState.bankDeposit * 1.02)
        playerState.bankLoan = Math.floor(playerState.bankLoan * 1.06)
    }

    if (playerState.privateLoan > 0) {
        playerState.privateLoan = Math.floor(playerState.privateLoan * 1.025)
        playerState.reputation = Math.max(-100, playerState.reputation - 3)
    }

    if (playerState.bankLoan > 0) {
        playerState.reputation = Math.max(-100, playerState.reputation - 2)
    }
}
```

### 3.4 每日消费扣除算法

```javascript
function nextDay(playerState) {
    let baseExpense = 80
    const fluctuation = Math.floor(Math.random() * 11) - 5
    let dailyExpense = baseExpense + fluctuation

    if (playerState.money >= dailyExpense) {
        playerState.money -= dailyExpense
    } else {
        const deficit = dailyExpense - playerState.money
        playerState.money = 0
        playerState.privateLoan += Math.ceil(deficit * 1.05)
        playerState.overdueDays++
    }
}
```

### 3.5 心情影响健康算法

```javascript
function applyMoodEffects(playerState) {
    if (playerState.mood < 40) {
        playerState.health -= 1
    }
    if (playerState.mood < 20) {
        playerState.health -= 2
    }
}
```

### 3.6 破产判定算法

```javascript
function checkBankruptcy(playerState) {
    const totalAssets = calculateTotalAssets(playerState)
    const totalDebt = playerState.bankLoan + playerState.privateLoan

    if (totalDebt > totalAssets * 1.5 && playerState.overdueDays >= 2) {
        return true
    }

    if (playerState.health <= 0 && playerState.money < 50) {
        return true
    }

    if (playerState.overdueDays >= 7) {
        return true
    }

    return false
}

function executeBankruptcy(playerState) {
```

### 3.7 结局判定算法

```javascript
function checkEnding(playerState) {
    if (playerState.health <= 0) return ENDINGS.HEALTH_COLLAPSE
    if (playerState.reputation <= 0) return ENDINGS.REPUTATION_DESTROYED
    if (checkBankruptcy(playerState)) return ENDINGS.BANKRUPTCY

    if (playerState.day > 180 && !playerState.purchasedHouse) {
        return ENDINGS.TIME_UP
    }

    if (playerState.purchasedHouse) {
        const totalDebt = playerState.bankLoan + playerState.privateLoan

        if (totalDebt > 0) {
            return ENDINGS.LOAN_DEBT
        }

        if (playerState.reputation >= 20 && playerState.health >= 40) {
            return ENDINGS.RESENTFUL_MARRIAGE
        }
        return ENDINGS.REGRET_BREAKUP
    }

    return null
}
```

## 十、通用组件规格

### 10.1 交互区域组件 (InteractiveArea)

#### 10.1.1 组件概述

交互区域组件是一种基于 Canvas 的可点击区域管理方案，用于在场景背景图上定义可交互的热点区域，支持文字标签、图标显示和点击事件处理。适用于房屋场景中的床、电脑等家具交互，以及其他需要背景图热点交互的场景。

#### 10.1.2 核心特性

| 特性 | 说明 |
|------|------|
| 百分比定位 | 使用相对坐标（0-1），自动适配不同屏幕尺寸 |
| 响应式渲染 | 文字、图标大小根据区域尺寸动态计算 |
| 视觉反馈 | 支持半透明高亮、边框、阴影等视觉效果 |
| 统一交互 | 集成 UIManager 点击检测系统 |
| 灵活配置 | 支持自定义样式、动画、回调函数 |

#### 10.1.3 数据结构定义

```javascript
// 交互区域配置
const InteractiveAreaConfig = {
    id: String,               // 区域唯一标识
    xPercent: Number,         // X轴位置（0-1，相对于屏幕宽度）
    yPercent: Number,         // Y轴位置（0-1，相对于屏幕高度）
    widthPercent: Number,     // 宽度（0-1，相对于屏幕宽度）
    heightPercent: Number,    // 高度（0-1，相对于屏幕高度）
    label: String,            // 显示文字标签
    icon: String,             // 图标（emoji或字符）
    action: Function,         // 点击回调函数
    style: AreaStyle          // 样式配置（可选）
};

// 区域样式配置
const AreaStyle = {
    bgColor: String,          // 背景色，默认 'rgba(255,255,255,0.1)'
    borderColor: String,      // 边框色，默认 'rgba(255,255,255,0.4)'
    borderWidth: Number,      // 边框宽度，默认 1.5
    borderRadius: Number,     // 圆角半径，默认 8
    textColor: String,        // 文字颜色，默认 '#FFFFFF'
    textShadow: Boolean,      // 是否启用文字阴影，默认 true
    iconSize: Number,         // 图标大小比例（相对于区域高度），默认 0.4
    fontSize: Number          // 字体大小（像素），默认自动计算
};

// 组件实例状态
const InteractiveAreaState = {
    areas: Map<String, InteractiveAreaConfig>,  // 区域配置映射
    hoveredArea: String,                        // 当前悬停区域ID
    pressedArea: String                         // 当前按下区域ID
};
```

#### 10.1.4 核心算法

**坐标转换算法（百分比转像素）**

```javascript
function getAreaPixels(area, screenWidth, screenHeight) {
    return {
        x: screenWidth * area.xPercent,
        y: screenHeight * area.yPercent,
        width: screenWidth * area.widthPercent,
        height: screenHeight * area.heightPercent
    };
}
```

**点击检测算法**

```javascript
function checkPointInArea(x, y, area, screenWidth, screenHeight) {
    const pos = getAreaPixels(area, screenWidth, screenHeight);
    return x >= pos.x && 
           x <= pos.x + pos.width && 
           y >= pos.y && 
           y <= pos.y + pos.height;
}
```

**响应式字体计算**

```javascript
function calculateFontSize(areaWidth, areaHeight, baseSize = 16) {
    const minDimension = Math.min(areaWidth, areaHeight);
    return Math.min(minDimension / 4, baseSize);
}
```

```javascript
    renderInteractiveAreas(renderer) {
        const ctx = renderer.ctx;
        const w = renderer.width;
        const h = renderer.height;
        
        Object.values(this.interactiveAreas).forEach(area => {
            const pos = getAreaPixels(area, w, h);
            const style = area.style || {};
            
            // 绘制背景
            ctx.fillStyle = style.bgColor || 'rgba(255,255,255,0.1)';
            ctx.beginPath();
            this.roundRectPath(ctx, pos.x, pos.y, pos.width, pos.height, 
                style.borderRadius || 8);
            ctx.fill();
            
            // 绘制边框
            ctx.strokeStyle = style.borderColor || 'rgba(255,255,255,0.4)';
            ctx.lineWidth = style.borderWidth || 1.5;
            ctx.stroke();
            
            // 绘制图标
            const iconSize = Math.min(pos.width, pos.height) * 
                (style.iconSize || 0.4);
            ctx.font = `${iconSize}px Arial`;
            ctx.textAlign = 'center';
            ctx.textBaseline = 'middle';
            ctx.fillStyle = style.textColor || '#FFFFFF';
            ctx.fillText(area.icon, pos.x + pos.width / 2, 
                pos.y + pos.height * 0.4);
            
            // 绘制文字
            const fontSize = calculateFontSize(pos.width, pos.height);
            ctx.font = `bold ${fontSize}px Arial, sans-serif`;
            
            if (style.textShadow !== false) {
                ctx.shadowColor = 'rgba(0,0,0,0.5)';
                ctx.shadowBlur = 4;
            }
            
            ctx.fillText(area.label, pos.x + pos.width / 2, 
                pos.y + pos.height * 0.75);
            ctx.shadowBlur = 0;
            
            // 注册点击区域
            this.game.uiManager.addButton(
                pos.x, pos.y, pos.width, pos.height,
                '', area.action, { bgColor: 'transparent' }
            );
        });
```

**适配原则：**
- 区域位置使用百分比，确保相对位置不变
- 文字大小根据区域最小边长动态计算，保证可读性
- 图标大小与区域大小成比例，保持视觉协调
- 最小点击区域不小于 44×44 像素（符合人机交互规范）

#### 10.1.8 最佳实践

1. **区域划分**：避免重叠，保持合理间距
2. **视觉层次**：重要区域使用更明显的边框/背景色
3. **文字简洁**：标签控制在 2-4 个字符
4. **图标选择**：使用通用 emoji 或自定义图标
5. **反馈明确**：点击后应有明确的视觉或弹窗反馈
6. **性能优化**：区域数量建议不超过 10 个，避免过多点击检测

## 十一、优化

### 11.1 资源加载优化

#### 11.1.1 存储架构

游戏采用双层存储架构，区分本地存储与文件系统：

| 存储类型 | API | 配额限制 | 适用场景 |
|---------|-----|---------|---------|
| 本地存储 | `wx.setStorage` / `wx.getStorageSync` | 50MB | 游戏存档、用户配置、临时状态 |
| 文件系统 | `wx.saveFile` / `wx.getFileSystemManager` | 200MB | 图片缓存、音频缓存 |

#### 11.1.2 缓存加载流程

资源加载采用**本地优先、云端兜底**的策略：

```
┌─────────────────────────────────────────────────────────┐
│                   资源加载流程                           │
├─────────────────────────────────────────────────────────┤
│  1. 检查本地缓存                                         │
│     ├─ 本地文件存在？                                    │
│     │   └─ 是 → 验证文件完整性                          │
│     │   │       ├─ 有效 → 直接使用本地文件               │
│     │   │       └─ 损坏 → 删除本地，重新下载             │
│     │   └─ 否 → 继续下一步                              │
│                                                         │
│  2. 尝试云端加载                                         │
│     ├─ 调用云函数获取临时链接                            │
│     ├─ 下载资源文件                                      │
│     ├─ 保存到本地文件系统                                │
│     └─ 更新缓存元数据                                    │
│                                                         │
│  3. 使用资源                                             │
│     └─ 返回可用的文件路径或内存对象                      │
└─────────────────────────────────────────────────────────┘
```

#### 11.1.3 缓存元数据管理

为追踪缓存状态，需要维护缓存元数据：

```javascript
const CacheMeta = {
    fileName: String,      // 缓存文件名
    cloudPath: String,     // 云端路径
    localPath: String,     // 本地保存路径
    size: Number,          // 文件大小(bytes)
    checksum: String,      // 完整性校验
    lastUpdate: Number,    // 最后更新时间戳
    version: String        // 资源版本号
};

// 缓存索引结构（存储在localStorage）
const CacheIndex = {
    lastSyncTime: Number,
    resources: {
        [resourceName]: CacheMeta
    }
}
```

#### 11.1.4 文件完整性验证

为防止缓存文件损坏，提供以下验证机制：

```javascript
// 方案1：文件大小验证
function validateBySize(filePath, expectedSize) {
    const fs = wx.getFileSystemManager()
    const stats = fs.statSync(filePath)
    return stats.size === expectedSize
}

// 方案2：简单存在性检查
function validateByExistence(filePath) {
    const fs = wx.getFileSystemManager()
    try {
        fs.statSync(filePath)
        return true
    } catch (e) {
        return false
    }
}
```

#### 11.1.5 缓存清理策略

| 策略 | 触发条件 | 说明 |
|------|---------|------|
| 自动清理 | 存储空间不足 | 清理最旧且非关键的资源 |
| 版本清理 | 资源更新 | 删除旧版本缓存 |
| 手动清理 | 用户主动触发 | 清除指定类型或全部缓存 |

#### 11.1.6 懒加载模式（可选）

对于非关键资源，可采用懒加载策略：

```
首次访问场景：
├── 检查本地缓存
├── 存在 → 直接使用
└── 不存在 → 后台下载 → 下载完成后通知UI更新

优势：
├── 减少首次加载时间
├── 按需占用存储空间
└── 用户只使用他玩的场景

注意：
├── 需要处理加载中的占位状态
└── 需要处理下载失败的重试逻辑
```

#### 11.1.7 技术实现要点

| 要点 | 说明 |
|------|------|
| 文件保存路径 | 使用 `wx.env.USER_DATA_PATH` 作为根目录 |
| 并发控制 | 限制同时下载的资源数量，避免占用过多带宽 |
| 错误处理 | 云端加载失败时提供友好提示，不阻塞游戏 |
| 进度反馈 | 大文件下载时提供加载进度（可选） |
| 内存管理 | 下载完成后及时释放内存，避免图片对象重复加载 |

#### 11.1.8 权限说明

| 权限类型 | 是否需要用户授权 | 说明 |
|---------|----------------|------|
| 文件读取 | ❌ 不需要 | 小程序目录内文件 |
| 文件写入 | ❌ 不需要 | 使用 wx.saveFile API |
| 网络请求 | ❌ 不需要 | 属于小程序内部行为 |
| 清除缓存 | ⚠️ 部分需要 | 用户手动清除需确认 |
