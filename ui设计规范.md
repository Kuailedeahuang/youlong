# 《吉卜力赛璐璐风模拟经营游戏UI设计规范》
## 一、设计核心原则（对齐美术风格）
1.  **「场景延伸」优先，拒绝界面割裂**
    UI 不是“盖在画面上的层”，而是**手绘场景的一部分**，必须完全遵循吉卜力背景的质感、线条、光影逻辑，不能出现任何写实/科技风元素。
2.  **信息极简，沉浸第一**
    所有常驻UI元素，必须做到“存在感极低但信息清晰”，不遮挡房间/场景内的核心互动区域，所有非必要信息默认隐藏。
3.  **100% 还原吉卜力赛璐璐质感**
    严格遵循「干净黑轮廓线+平涂色块+柔和水彩晕染+无硬阴影」的美术规则，拒绝高对比、强发光、写实纹理。

## 二、色彩系统（严格匹配场景配色）
### 1. 主色调（从你的场景提取）
| 色彩用途 | 色值参考 | 风格要求 |
| :--- | :--- | :--- |
| UI 基底色（面板/按钮背景） | 浅天蓝色 `#E0F0FF`、浅米木色 `#FFF5E6` | 低饱和、高明度，带轻微水彩晕染质感，无纯黑/纯白 |
| 强调色（关键按钮/高亮） | 暖黄/浅橙 `#FFE080`（道路标线同款） | 仅用于关键操作，占比≤5%，避免大面积使用 |
| 禁用色 | 纯黑、纯白、高饱和红/绿/蓝、强对比色 | 所有元素的对比度控制在 WCAG AA 标准，同时保持柔和感 |

### 2. 文字色彩规范
- 正文文字：深灰蓝 `#4A5568` 或 深木棕 `#5D4037`，避免纯黑
- 标题/关键信息：主色加深20%（如 `#87CEEB` 加深为 `#5A9BD5`），可加1px 同色系细描边
- 禁用：高饱和彩色文字、纯黑文字、带强烈外发光的文字

## 三、图形与轮廓规范（核心！决定风格统一度）
1.  **线条规范（完全对齐场景轮廓）**
    - 所有UI元素（按钮、面板、图标）必须使用**1-2px 干净黑色轮廓线**，和场景中家具、建筑的线条粗细、质感完全一致
    - 线条不能有毛边、渐变、模糊效果，保持清晰锐利，还原赛璐璐动画的轮廓感
    - 禁用：无轮廓线、渐变轮廓线、粗重黑边（≥3px）、圆角过度圆润/过度尖锐

2.  **形状与质感规范**
    - 所有UI元素采用**大圆角设计**，圆角半径为元素高度的15%-20%，和场景中门窗、家具的圆角风格统一
    - 面板/按钮背景为**平涂色块+极轻微水彩晕染纹理**，无渐变、无强阴影、无写实纹理（如木纹、金属反光）
    - 阴影：仅使用「柔和、低饱和的同色系投影」，投影透明度≤20%，无硬边阴影、无模糊阴影，模拟吉卜力动画里“软乎乎的自然光投影”
    - 禁用：直角元素、复杂几何形状、3D立体效果、强烈投影、玻璃/金属反光质感

3.  **图标规范（和场景融为一体）**
    - 风格：纯平涂赛璐璐风图标，带1px 黑色轮廓线，无渐变、无立体效果
    - 设计逻辑：提取场景中的元素做简化，比如：
      - 背包 → 简化的手绘小木箱（和房间里的家具质感一致）
      - 建造 → 手绘小锤子（平涂+黑轮廓）
      - 金钱 → 简化的硬币图标（暖黄色平涂+轮廓线）
    - 禁用：写实图标、科技风线性图标、拟物图标、渐变图标

## 四、排版与字体规范
1.  **字体选择**
    - 优先使用**圆润无衬线字体**，如「思源柔黑体、站酷快乐体、汉仪粗圆体」，避免宋体、黑体等棱角分明的字体
    - 所有文字可加1px 同色系细描边，保证在不同背景下的可读性，同时不破坏柔和感
    - 禁用：衬线字体、科技感字体、手写体（除非是标题/装饰文字）、带强烈效果的字体（如立体字、描边过粗的文字）

2.  **排版规则**
    - 行高：1.5倍，字间距+50，保证文字宽松舒适，符合治愈系氛围
    - 标题：字号比正文大20%-30%，颜色用加深的主色，可加轻微的文字投影（透明度≤15%）
    - 正文：字号统一，颜色用深灰蓝/深木棕，避免纯黑
    - 禁用：文字拥挤、多行密集文字、不同字体混用、强烈的文字特效

## 五、布局与层级规范（保护场景沉浸感）

### 场景页面三段式布局

场景页面采用**三段式布局**，所有交互通过地图组件完成：

**1. 顶部信息栏**
- **内容**：天数、金币、银行净额、私人欠款、登录状态
- **设计**：半透明深色背景条，带底部分隔线
- **天数显示**：📅 图标 + 第X天 / 总天数
- **金币显示**：💰 图标 + 大字号数字（金色突出）
- **银行净额**：🏦 图标 + 净额（绿色正数/红色负数）
- **私人欠款**：🚨 图标 + 欠款金额（红色警示，有欠款时显示）
- **登录状态**：✓ 已登录 / ✗ 未登录（紧凑显示在左侧）

**2. 中间场景区域**
- **内容**：场景背景图 + 地图组件
- **设计**：占据主要画面空间，提供沉浸式体验
- **交互**：点击地图特定区域跳转到对应场景或打开功能界面
- **场景切换**：完全通过地图点击完成，无Tab栏/按钮切换

**3. 底部属性面板**
- **内容**：角色属性面板（健康、精力、心情、名誉）
- **设计**：半透明面板背景，2x2网格布局
- **属性展示**：
  - ❤️ 健康 - 绿色进度条
  - ⚡ 精力 - 蓝色进度条（高亮显示）
  - 😊 心情 - 粉色进度条
  - ⭐ 名誉 - 紫色进度条
- **进度条**：带背景色和填充色，低数值时自动变色警示

### 通用布局原则
- 所有常驻UI元素不遮挡场景核心互动区域
- 非必要信息默认隐藏或折叠
- 弹窗/菜单居中显示，不遮挡地图交互点
- 面板带1px黑色轮廓线，大圆角设计
- 禁用：全屏弹窗、遮挡核心互动元素的弹窗

### 场景内互动UI
- 物品互动提示：仅使用「迷你手绘图标+微弱光晕」，不使用文字提示
- 对话气泡：浅米色/浅蓝色圆角气泡，带1px轮廓线，停留时间≤3秒
- 禁用：常驻文字提示、大尺寸互动图标

## 六、交互与动效规范（完全贴合吉卜力动画质感）
1.  **动效核心原则**：温柔、克制、无干扰，拒绝强烈的震动、闪光、弹跳效果
2.  **按钮交互反馈**
    - hover/点击：轻微缩放（95%-100%）+ 颜色加深10%，无其他特效
    - 禁用：强烈弹跳、发光、震动、粒子特效
3.  **弹窗动效**
    - 淡入淡出：时长300ms，无缩放/位移，仅透明度变化
    - 场景切换：溶解/淡入淡出过渡，时长500ms，无转场特效
4.  **飘字/提示动效**
    - 收入/支出提示：暖黄色/浅橙色文字，从物品位置向上缓慢飘出，透明度从100%降到0%，时长2秒，无缩放
    - 状态提示（如“冰箱空了”）：迷你气泡从物品上方弹出，停留3秒后自动消失，无震动
5.  **场景内互动动效**
    - 物品互动：轻微的手绘抖动（1-2帧），和吉卜力动画中物品的轻微晃动质感一致，无其他特效
    - 客人交互：对话气泡淡入淡出，客人动作保持和场景一致的手绘帧动画，无额外特效

## 七、禁用清单（红线！避免风格割裂）
❌ 禁止使用纯黑、纯白、高饱和对比色
❌ 禁止使用科技风、写实风、拟物风的图标/字体/特效
❌ 禁止使用强阴影、硬边投影、渐变、立体效果
❌ 禁止使用常驻大型UI、全屏弹窗、遮挡核心场景的元素
❌ 禁止使用强烈的动效、闪光、震动、粒子特效
❌ 禁止混用多种线条粗细、多种字体、多种风格的元素

## 八、快速落地检查清单
每次做UI元素时，对照这几点检查，就能保证和场景100%统一：
1.  有没有1-2px 干净的黑色轮廓线？
2.  背景是不是平涂色块+轻微水彩晕染，无强阴影/渐变？
3.  颜色是不是低饱和、高明度，和场景主色呼应？
4.  文字是不是圆润无衬线字体，无纯黑/高饱和彩色？
5.  动效是不是温柔克制，没有强烈特效？
6.  有没有遮挡场景内的核心互动元素？
